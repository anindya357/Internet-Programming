🏋️ FitTrack CUET - Testing Guide

Welcome to FitTrack CUET!

This guide will help you test all features of the gymnasium management system. The application is a fully functional frontend prototype with simulated data storage using LocalStorage.

🚀 Quick Start

Start testing by visiting any of these pages:

🔐 Login Page 📝 Register 📊 Dashboard ⚙️ Admin Panel

📋 Testing Checklist

1. Authentication System

Test Steps:
  • Go to index.html
  • Enter any Student ID (e.g., 1801001)
  • Enter any password
  • Click "Login" button
  • Test password visibility toggle
  • Try "Register here" link
Expected Result: Successful login with redirect to dashboard

2. User Registration

Test Steps:
  • Go to register.html
  • Fill in all required fields
  • Test password validation (min 8 chars, uppercase, lowercase, numbers)
  • Confirm password match
  • Accept terms and conditions
  • Click "Create Account"
Expected Result: Account created with redirect to login

3. Dashboard Features

Welcome Section

  • Personalized greeting
  • Quick action buttons

Gym Status

  • Operating hours
  • Current capacity
  • Environment info

Statistics

  • Total workouts
  • Monthly count
  • Current streak

Equipment Catalog

  • Filter by category
  • Availability status
  • Equipment details

4. Workout History

Test Steps:
  • Click "Log New Workout" button
  • Fill in workout details (date, type, duration)
  • Add multiple exercises with sets/reps/weights
  • Select intensity level
  • Add notes (optional)
  • Save workout
  • Test filters (date range, type, search)
  • View and delete workouts

5. AI Gym Instructor

Test Features:
  • Quick prompt buttons for common questions
  • Type custom fitness questions
  • Test responses for:
    • Workout plans
    • Equipment usage
    • Injury prevention
    • Warm-up routines
    • Progressive overload tips
  • Check typing indicator animation
  • Test message formatting (bold, lists)

6. Medical Profile

Test Steps:
  • Fill in basic health info (age, height, weight)
  • Check automatic BMI calculation
  • Select blood group
  • Check chronic conditions
  • Enter past injuries and medications
  • Toggle allergy information
  • Set fitness goals and limitations
  • Add emergency contact
  • Save profile (data persists in LocalStorage)

7. Diet Plan

Test Features:
  • View nutrition goals (calories, macros)
  • Set diet preferences:
    • Diet type (balanced, high-protein, etc.)
    • Meals per day (3-6)
    • Target calories
  • Select dietary restrictions
  • View today's meal plan with:
    • Meal times
    • Food items
    • Calorie counts
    • Macro breakdown
  • Switch between days in weekly overview
  • Generate new diet plan

8. Rules & Safety

Test Elements:
  • Read safety highlights
  • Review 10 general gym rules
  • Check safety guidelines
  • View injury prevention tips
  • Review equipment usage guidelines
  • See prohibited activities
  • Acknowledge rules (checkbox + button)
  • Check acknowledgment persistence

9. Admin Dashboard

Test Features:
  • View key metrics (members, active users, equipment)
  • Toggle gym status (open/closed)
  • Monitor current capacity
  • Post announcements
  • View recent activity table
  • Manage equipment:
    • Search equipment

💾 Data Persistence Testing

Test LocalStorage:
  • Login → Check userData in localStorage
  • Create workout → Refresh page → Check if workout exists
  • Save medical profile → Logout → Login → Check if data persists
  • Update diet preferences → Navigate away → Return → Check saved preferences
  • Acknowledge rules → Close browser → Reopen → Check acknowledgment status
✅ Testing Complete!

If all features work as expected, the frontend is ready for backend integration.

🐛 Known Limitations

  • No real authentication (accepts any credentials)
  • Data stored in browser only (not synced across devices)
  • AI responses are pre-defined (not real AI)
  • No real-time updates (refresh needed for some changes)
  • Limited data validation

🔧 Browser Developer Tools

Use these tools for testing:

  • Console (F12): Check for JavaScript errors
  • Application tab: View LocalStorage data
  • Network tab: Monitor resource loading
  • Responsive mode: Test different screen sizes
  • Lighthouse: Performance and accessibility audit

📞 Support

For issues or questions:

  • Check browser console for errors
  • Clear LocalStorage if data seems corrupted
  • Try different browser or incognito mode
  • Review code comments in source files
